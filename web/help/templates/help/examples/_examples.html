Remarks:

  • Instead of individually dragging the elements to their respective section, the quicker way is to create the sections and then in the Form View select all elements of a section and set their respective section in the properties in one go. Then all it's necessary is to order them.
  • Checkboxes and Radios groups require to be dragged to their respective sections.
  • Since the headers and footer have no editable or user interactable elements, they don't appear in the List View and therefore cannot be included in any section.
  • It's always better to waste some time giving proper names to each element, to prevent confusion later on - only editable / interactable elements can be placed in a section.
  • Use the Name property for identifying the different elements in the Designer and use the Label/Text to set what will be presented to the end user in the List View.


Setting up the Events/Actions


We will define the following E/As:

# Event Action Class Action Description
1 Insert Personal ID Table Query Retrieve the personal Name, Address, Telephone and Email
2 Insert Target Fiscal Number Database Query Retrieve the Target Name, Address, Telephone and Email
3 Check / Uncheck Drawing? checkbox Status Enable / Disable Drawing Description and Drawing Area (*)
4 Check / Uncheck Container? checkbox Status Enable / Disable Container Group (*)
5 Check / Uncheck Other radio of the Container Group Status Enable / Disable Container Other (*)
6 Check / Uncheck Other radio of the Container Group Required Required / Not Required Container Other (***)
7 Check / Uncheck Signature #2? Visibility Show / Hide Signature #2 (**)
8 Check / Uncheck Signature #3? Status Cross / Uncross Signature #3
9 Fill First Name and Last Name Append Full Name = First Name #-# Last Name

Notes:

  • (*) - Disabled by default
  • (**) - Enabled by default
  • (***) - Not required by default

Remarks:

  • Use the Required property of an element or of the Group to set whether a value or selection is mandatory.
  • Use the Enabled property of an element to set whether the element is enabled by default.
  • Use the Visible property of an element to set whether the element is visible by default.
  • Don't forget to create the reverse E/A, when available.
  • Despite in this example we create the entire form and then set its sections, groups and E/As, it's good pratice to as soon all elements corresponding to a visual section are layed out, to set its section, all its groups and its E/As. This avoid the time consuming task of searching specific elements in what could end up being a very long lists of elements.
  • Cloning similar E/As allows for a faster E/A creation, specially when we have an high number of elements.

Forms manager


Once the form is ready to be used, we need to set its status from EDITABLE to IN USE. Before changing its status, make sure the form is ready to be used, otherwise, only an administrator possesses the privileges to revert the IN USE status back to EDITABLE, or as an alternative, create a clone of the form.
